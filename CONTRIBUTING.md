# Contributing

Thank you for contributing to this project.

## Documentation Must Stay In Sync

When you change code, update documentation in the same pull request if the change affects behavior, structure, or user guidance.

Update these files when relevant:

- `README.md`: user-facing setup, run steps, and feature behavior.
- `docs/ARCHITECTURE.md`: module structure, responsibilities, or data flow.
- `docs/DEPENDENCIES.md`: imports, package dependencies, or module coupling.
- `docs/START_HERE.md`: beginner onboarding steps.

Typical triggers for documentation updates:

- New user input options, chart behavior, or CLI/Streamlit workflow changes.
- New modules, renamed functions, or changed responsibilities between modules.
- Added/removed dependencies in `pyproject.toml`.
- Changes to file layout, outputs, or how results should be interpreted.

If no documentation updates are needed, explain why in the pull request description.
